GIT MERGE

Merging takes the contents of a source branch and integrates them with a target branch. In this process, only the target branch is changed. The source branch history remains the same.

1. git merge --abort : It will abort the merge process and try to reconstruct the pre-merge state.

2. git merge --continue : It is run after the merge has resulted in conflicts.

Options in GIT Merge

--commit --no-commit

--edit -e --no-edit

--ff --no-ff --ff-only

--signoff --no-signoff

--stat -n --no-stat

--squash --no-squash

--abort

--quit

--continue
